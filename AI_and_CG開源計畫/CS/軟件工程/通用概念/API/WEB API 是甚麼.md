## 可以將 Web API 視為 客戶端 和 網路資源 之間的溝通接口

![[Pasted image 20260419050433.png]]

想像現在自己正坐在一家餐廳裡，我們的 **顧客** 需要點餐，會先查看菜單，然後告訴 **服務生**，我們要想吃什麼，然後就會把 **點餐單** 送去 **廚房** ，廚房這時候就會去確認這道菜存不存在、有沒有**庫存**、要不要記錄這筆訂單，然後開始 **烹製料理**，最後由 **服務生** 將完成的**料理**送回桌上

| 餐廳角色 | 系統角色          | 在做什麼           |
| ---- | ------------- | -------------- |
| 顧客   | 前端（Frontend）  | 操作畫面、點擊按鈕、輸入資料 |
| 點餐單  | Request（請求）   | 帶有格式跟訊息的資料文件   |
| 服務生  | API           | 負責傳送請求與回應      |
| 廚房   | 後端（Backend）   | 處理請求、驗證資料      |
| 庫存   | 資料庫（Database） | 儲存資料（帳號、訂單）    |
| 料理   | 回應（Response）  | 回傳結果給前端        |
>
> 前端負責顯示，API負責傳資料，後端負責處理，資料庫負責存資料
