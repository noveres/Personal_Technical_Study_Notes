
# 接案報價系統 V1

**定位：給接案者 / 小型工作室使用的超快速報價工具**

核心價值只有一句：

**3 分鐘內完成一份可交付給客戶的報價單。**

---
 
V1 只做這 4 件事：

1. **客戶管理**(登入註冊)
    
2. **報價單建立**
    
3. **報價單匯出 / 列印**

- 可登入
    
- 可新增客戶
    
- 可建立報價單
    
- 可新增多筆項目
    
- 系統自動計算金額
    
- 可預覽
    
- 可匯出 PDF


---

# 二、目標使用流程

使用者進來後，流程應該是：

**建立客戶 → 建立報價單 → 加入報價項目 → 預覽 → 匯出 PDF**

你要讓整個過程很像填表，而不是像在操作 ERP。

我建議你直接把 V1 的計算模型設計成下面這樣。

---

## 1. 先分清楚 3 種報價模式

### A. 稅前報價

最常見。

意思是：

- 使用者輸入的是 **未稅單價**
    
- 系統最後再加稅
    

公式：

```text
小計 = 數量 × 未稅單價
未稅總額 = 所有小計加總
稅額 = 未稅總額 × 稅率
含稅總額 = 未稅總額 + 稅額
```

例子：

```text
網站設計：1 × 20,000 = 20,000
RWD 切版：1 × 12,000 = 12,000

未稅總額 = 32,000
稅額 = 32,000 × 5% = 1,600
含稅總額 = 33,600
```

---

### B. 稅後報價

有些接案者報價時，腦中想的是：

**「我就是要報 30,000，客戶看到也就是 30,000。」**

這時輸入的是 **含稅價**，系統要反推未稅。

公式：

```text
未稅金額 = 含稅金額 ÷ (1 + 稅率)
稅額 = 含稅金額 - 未稅金額
```

例子：

```text
含稅總價 = 31,500
稅率 = 5%

未稅金額 = 31,500 ÷ 1.05 = 30,000
稅額 = 1,500
```

這模式很重要，因為很多接案者跟客戶談價格時，談的是「最後總價」，不是未稅價。

---

### C. 免稅 / 不計稅

你一定要保留這種模式。

原因不是所有人都一定開發票，也不是每張報價都一定要算營業稅；而且財政部對不同情況也有不同規範，例如部分交易可能有零稅率或其他適用情形。V1 不要硬做太細，但資料模型要留彈性。([財政部](https://www.mof.gov.tw/singlehtml/384fb3077bb349ea973e7fc6f13b6974?cntId=f43f5af76bd7483eb6497e005b0a1e81&utm_source=chatgpt.com "網紅課稅有規定，國稅局報乎你知！"))

公式：

```text
未稅總額 = 含稅總額 = 所有項目加總
稅額 = 0
```

---

## 2. 我建議你的資料表加這幾個欄位

你原本的 `quotations` 表，建議補成這樣：

```sql
tax_mode VARCHAR(20) NOT NULL DEFAULT 'EXCLUDED',
tax_rate DECIMAL(5,2) NOT NULL DEFAULT 5.00,
rounding_mode VARCHAR(20) NOT NULL DEFAULT 'ROUND',
discount_type VARCHAR(20) NOT NULL DEFAULT 'NONE',
discount_value DECIMAL(12,2) NOT NULL DEFAULT 0,
```

### 欄位意思

`tax_mode`

- `EXCLUDED`：稅外，加稅
    
- `INCLUDED`：稅內，反推
    
- `NONE`：不計稅
    

`tax_rate`

- 先預設 5
    
- 但不要寫死
    

`rounding_mode`

- `ROUND`：四捨五入
    
- `FLOOR`：無條件捨去
    
- `CEIL`：無條件進位
    

`discount_type`

- `NONE`
    
- `AMOUNT`：整單折扣金額
    
- `PERCENT`：整單折扣百分比
    

`discount_value`

- 折多少
    

---

## 3. 報價單的正確計算順序

我建議你統一採這個順序，之後才不會亂：

### Step 1：先算每個項目小計

```text
項目小計 = 數量 × 單價
```

### Step 2：加總成商品總額

```text
items_total = 所有項目小計加總
```

### Step 3：套用折扣

```text
discounted_total = items_total - discount
```

如果是百分比折扣：

```text
discount_amount = items_total × 折扣百分比
discounted_total = items_total - discount_amount
```

### Step 4：根據 tax_mode 算稅

#### 稅外

```text
subtotal = discounted_total
tax_amount = subtotal × tax_rate
total_amount = subtotal + tax_amount
```

#### 稅內

```text
total_amount = discounted_total
subtotal = total_amount ÷ (1 + tax_rate)
tax_amount = total_amount - subtotal
```

#### 不計稅

```text
subtotal = discounted_total
tax_amount = 0
total_amount = subtotal
```

### Step 5：最後做 rounding

不要每個步驟都亂四捨五入。  
**建議只在最終顯示金額，或稅額切分時做一次。**

---

## 4. 你的系統要支援「單價是未稅還含稅」

這很關鍵。

因為使用者可能會說：

## 1. 先分清楚 3 種報價模式

### A. 稅前報價

最常見。

意思是：

- 使用者輸入的是 **未稅單價**
    
- 系統最後再加稅
    

公式：

```text
小計 = 數量 × 未稅單價
未稅總額 = 所有小計加總
稅額 = 未稅總額 × 稅率
含稅總額 = 未稅總額 + 稅額
```

例子：

```text
網站設計：1 × 20,000 = 20,000
RWD 切版：1 × 12,000 = 12,000

未稅總額 = 32,000
稅額 = 32,000 × 5% = 1,600
含稅總額 = 33,600
```

---

### B. 稅後報價

有些接案者報價時，腦中想的是：

**「我就是要報 30,000，客戶看到也就是 30,000。」**

這時輸入的是 **含稅價**，系統要反推未稅。

公式：

```text
未稅金額 = 含稅金額 ÷ (1 + 稅率)
稅額 = 含稅金額 - 未稅金額
```

例子：

```text
含稅總價 = 31,500
稅率 = 5%

未稅金額 = 31,500 ÷ 1.05 = 30,000
稅額 = 1,500
```

這模式很重要，因為很多接案者跟客戶談價格時，談的是「最後總價」，不是未稅價。

---

### C. 免稅 / 不計稅

你一定要保留這種模式。

原因不是所有人都一定開發票，也不是每張報價都一定要算營業稅；而且財政部對不同情況也有不同規範，例如部分交易可能有零稅率或其他適用情形。V1 不要硬做太細，但資料模型要留彈性。([財政部](https://www.mof.gov.tw/singlehtml/384fb3077bb349ea973e7fc6f13b6974?cntId=f43f5af76bd7483eb6497e005b0a1e81&utm_source=chatgpt.com "網紅課稅有規定，國稅局報乎你知！"))

公式：

```text
未稅總額 = 含稅總額 = 所有項目加總
稅額 = 0
```

## 3. 報價單的正確計算順序

我建議你統一採這個順序，之後才不會亂：

### Step 1：先算每個項目小計

```text
項目小計 = 數量 × 單價
```

### Step 2：加總成商品總額

```text
items_total = 所有項目小計加總
```

### Step 3：套用折扣

```text
discounted_total = items_total - discount
```

如果是百分比折扣：

```text
discount_amount = items_total × 折扣百分比
discounted_total = items_total - discount_amount
```

### Step 4：根據 tax_mode 算稅

#### 稅外

```text
subtotal = discounted_total
tax_amount = subtotal × tax_rate
total_amount = subtotal + tax_amount
```

#### 稅內

```text
total_amount = discounted_total
subtotal = total_amount ÷ (1 + tax_rate)
tax_amount = total_amount - subtotal
```

#### 不計稅

```text
subtotal = discounted_total
tax_amount = 0
total_amount = subtotal
```

### Step 5：最後做 rounding

不要每個步驟都亂四捨五入。  
**建議只在最終顯示金額，或稅額切分時做一次。**
